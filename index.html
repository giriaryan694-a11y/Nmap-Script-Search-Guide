Nmap NSE Script Search Guide

Introduction: Nmap's scripting engine (NSE) enhances scanning by adding automation, detection, exploitation, and enumeration capabilities. NLP scripts are categorized based on their purpose, enabling penetration testers and students to quickly select the right script for a target. This guide acts like clean notes for learning how to search and explore scripts efficiently.

Types of Nmap Scripts

Type Description
Discovery Used for identifying hosts, services, and basic network info. Helps in early recon.
Vulnerability Checks for known vulnerabilities, misconfigurations, outdated versions, etc.
Brute Attempts login or password brute force attack on various services.

This page provides simple and powerful ways to search, filter, and explore Nmap NSE scripts using cat and grep. Ideal for penetration testers, researchers, and students learning advanced recon techniques.

1. View All Script Names

ls /usr/share/nmap/scripts/

Check script.db for full list:

cat /usr/share/nmap/scripts/script.db

2. Search Scripts by Type (Category)

Discovery scripts

grep -R "categories = .*discovery" /usr/share/nmap/scripts/

Vulnerability scripts

grep -R "vuln" /usr/share/nmap/scripts/

Brute force scripts

grep -R "brute" /usr/share/nmap/scripts/

3. Using cat + grep Together

Search inside script.db

cat /usr/share/nmap/scripts/script.db | grep categories

Search for vuln scripts

cat /usr/share/nmap/scripts/script.db | grep "vuln"

Search for default scripts

cat /usr/share/nmap/scripts/script.db | grep "default"

4. Pro-Level Filters

SMB-related scripts

grep -Ri "smb" /usr/share/nmap/scripts/

SQL Injection scripts

grep -Ri "sql" /usr/share/nmap/scripts/

SSL/TLS scripts

grep -Ri "ssl" /usr/share/nmap/scripts/

How NSE Works Internally

Nmap's NSE engine loads scripts written in Lua, assigns them categories, and runs them during specific scan phases. Scripts interact with targets using Nmap libraries that allow networking, protocol parsing, brute forcing, and more.

Main Components:

  • Script: Written in Lua and stored in /usr/share/nmap/scripts/.
  • Libraries: Provide reusable functions for SMB, HTTP, DNS, etc.
  • Categories: Define when and how scripts run.

Common NSE Categories

  • safe – Non-intrusive scripts.
  • intrusive – Might affect target stability.
  • vuln – Checks for vulnerabilities.
  • auth – Authentication-related checks.
  • default – Runs in -sC scan.
  • brute – Performs brute force attacks.
  • discovery – Finds hosts and services.

Create Your First NSE Script

description = "Simple NSE Script"

author = "Aryan"

categories = {"safe"}

portrule = function(host, port) return port.protocol == "tcp" and port.state == "open" end

action = function(host, port) return "Test scan on " .. host.ip end
